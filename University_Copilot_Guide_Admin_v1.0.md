## 1. 서론

### 1.1 GitHub Copilot이란 무엇인가?
GitHub Copilot은 Microsoft와 GitHub가 공동 개발한 AI 기반의 '지능형 업무 파트너'입니다. 초기에는 소프트웨어 개발자를 위한 코드 자동 완성 도구로 시작되었으나, 2025년 현재는 GPT-5.1, Gemini 3 Pro 등 최신 AI 모델을 탑재하여 문서 작성, 데이터 분석, 업무 자동화 등 다양한 영역에서 활용되는 강력한 도구로 진화하였습니다.

단순히 사용자의 명령을 수행하는 것을 넘어, 사용자가 작성하려는 문서의 맥락(Context)을 이해하고, 다음에 올 내용을 미리 예측하여 제안하며, 복잡한 업무 흐름을 간소화하는 데 도움을 줍니다.

### 1.2 대학 행정직원에게 왜 필요한가?
대학 행정 업무는 기획, 예산, 학사 관리, 연구 지원 등 방대한 양의 문서를 다루는 지식 집약적 활동입니다. GitHub Copilot은 다음과 같은 측면에서 행정직원의 업무 효율을 혁신적으로 높일 수 있습니다.

*   **문서 작성 시간 단축:** 공문, 보고서, 이메일 등의 초안을 빠르게 생성하고, 반복적인 문구를 자동 완성하여 작성 시간을 획기적으로 줄여줍니다.
*   **업무 정확도 향상:** 방대한 규정집이나 과거 회의록을 AI가 실시간으로 참조하여, 규정에 어긋나지 않는 정확한 문서 작성을 지원합니다.
*   **단순 반복 업무 자동화:** 엑셀 데이터 정리, 파일 형식 변환 등 번거로운 단순 반복 업무를 AI에게 위임하여, 행정직원은 보다 창의적이고 전략적인 업무에 집중할 수 있습니다.

### 1.3 이 가이드의 목표
본 가이드는 IT나 프로그래밍에 대한 지식이 전혀 없는 대학 행정직원을 대상으로 합니다. 복잡한 기술 용어는 최대한 배제하고, 대학 행정 현장에서 즉시 활용할 수 있는 실무 중심의 내용을 담았습니다. Visual Studio Code(VS Code) 설치부터 Git을 이용한 버전 관리, 그리고 GitHub Copilot의 최신 기능을 활용하는 방법까지 단계별로 안내합니다.

## 제1장: 업무 환경 구축

AI 코딩 어시스턴트를 효과적으로 활용하기 위해서는 먼저 적절한 디지털 업무 환경을 구축해야 합니다. 본 장에서는 가장 강력한 문서 작성 도구인 Visual Studio Code와 버전 관리 시스템인 Git, 그리고 GitHub Copilot을 설치하고 설정하는 과정을 다룹니다.

### 1.1 Visual Studio Code (VS Code) 설치 및 설정

#### 1.1.1 VS Code란?
Visual Studio Code(이하 VS Code)는 Microsoft에서 개발한 무료 텍스트 편집기입니다. 흔히 사용하는 '메모장'이나 '한글'과 비슷해 보일 수 있지만, 전 세계 개발자와 지식 근로자들이 가장 애용하는 도구입니다. 가볍고 빠르며, 수많은 '확장 프로그램(Extensions)'을 통해 기능을 무한히 확장할 수 있다는 장점이 있습니다. 행정직원에게는 단순한 문서 편집기를 넘어, AI와 협업하는 '디지털 워크스테이션' 역할을 수행합니다.

#### 1.1.2 설치 방법
1.  **다운로드:** 웹 브라우저에서 [Visual Studio Code 공식 홈페이지](https://code.visualstudio.com/)에 접속합니다.
2.  **설치 파일 실행:** 메인 화면의 'Download' 버튼을 클릭하여 운영체제(Windows 또는 macOS)에 맞는 설치 파일을 다운로드하고 실행합니다.
3.  **설치 진행:** 설치 마법사의 안내에 따라 설치를 진행합니다. 특별한 설정 변경 없이 '다음(Next)'을 클릭하여 기본값으로 설치해도 무방합니다.

#### 1.1.3 기본 설정 (한국어 패치)
VS Code는 기본적으로 영어로 설치되지만, 한국어 언어 팩을 설치하여 편리하게 사용할 수 있습니다.

1.  VS Code를 실행합니다.
2.  왼쪽 사이드바에서 블록 모양의 **'Extensions(확장)'** 아이콘을 클릭합니다. (단축키: `Ctrl+Shift+X` 또는 `Cmd+Shift+X`)
3.  검색창에 `Korean`을 입력합니다.
4.  **'Korean Language Pack for Visual Studio Code'** 항목을 찾아 **'Install'** 버튼을 클릭합니다.
5.  설치가 완료되면 오른쪽 하단에 나타나는 **'Change Language and Restart'** 버튼을 클릭하여 VS Code를 재시작합니다. 이제 모든 메뉴가 한국어로 표시됩니다.

### 1.2 Git 설치 및 기본 개념

#### 1.2.1 Git이란?
Git(깃)은 파일의 변경 이력을 관리하는 '버전 관리 시스템'입니다. 쉽게 말해, 문서의 타임머신과 같습니다. 우리가 흔히 문서를 수정할 때 만드는 `보고서_최종.hwp`, `보고서_진짜최종.hwp`, `보고서_최종_수정본.hwp`와 같은 파일명 지옥에서 벗어나게 해주는 도구입니다.

#### 1.2.2 왜 행정 문서에 Git이 필요한가?
*   **이력 추적:** 누가, 언제, 어떤 내용을 수정했는지 정확하게 기록됩니다.
*   **복구 용이성:** 문서 작성 중 실수를 하거나 내용이 삭제되더라도, 언제든지 과거의 특정 시점으로 되돌릴 수 있습니다.
*   **협업 효율화:** 여러 사람이 하나의 문서를 동시에 작업하더라도, 변경 사항을 안전하게 합치고 충돌을 관리할 수 있습니다.

#### 1.2.3 설치 및 초기 설정
**Windows 사용자:**
1.  [Git for Windows](https://git-scm.com/download/win) 사이트에서 설치 파일을 다운로드하여 실행합니다.
2.  설치 과정에서 나오는 옵션들은 기본값(Default)을 유지하며 'Next'를 눌러 설치를 완료합니다.

**macOS 사용자:**
1.  터미널(Terminal) 앱을 실행합니다 (`Cmd+Space` 후 'Terminal' 검색).
2.  `git --version` 명령어를 입력합니다.
3.  설치되어 있지 않다면 자동으로 설치 팝업이 뜹니다. 안내에 따라 설치합니다. (또는 [Git 공식 사이트](https://git-scm.com/download/mac)에서 설치)

**사용자 정보 등록 (필수):**
설치 후, 터미널(VS Code 내의 터미널도 가능)에서 다음 두 줄의 명령어를 입력하여 본인의 정보를 등록해야 합니다. 이 정보는 문서 수정 이력에 기록됩니다.

```bash
git config --global user.name "홍길동"
git config --global user.email "gildong@deu.ac.kr"
```

### 1.3 GitHub Copilot 도입

#### 1.3.1 라이선스 준비
GitHub Copilot을 사용하기 위해서는 유료 구독 라이선스가 필요합니다.
*   **개인 사용자:** 월 $10 (또는 연 $100)
*   **대학 기관:** GitHub Enterprise 또는 Copilot Business 플랜을 통해 교직원에게 라이선스를 배포할 수 있습니다.
*   **참고:** 인증된 교육 기관의 학생 및 교원은 GitHub Education 혜택을 통해 무료로 사용할 수도 있습니다.

#### 1.3.2 VS Code 확장 프로그램 설치
2025년 11월 현재, GitHub Copilot의 기능은 **'GitHub Copilot'**이라는 하나의 통합된 확장 프로그램으로 제공됩니다.

1.  VS Code의 **'확장(Extensions)'** 탭을 엽니다.
2.  `GitHub Copilot`을 검색합니다.
3.  Microsoft(GitHub)가 배포한 공식 확장 프로그램을 찾아 **'설치(Install)'**합니다.
4.  설치 후 VS Code 우측 하단 또는 알림창을 통해 GitHub 계정 로그인을 진행합니다.
5.  로그인이 완료되고 상태 표시줄에 Copilot 아이콘이 활성화되면 사용할 준비가 완료된 것입니다.

## 제2장: VS Code 기초

프로그래밍 경험이 없는 행정직원에게 VS Code의 화면은 다소 낯설게 느껴질 수 있습니다. 하지만 문서 작성에 필요한 핵심 기능만 익히면, 기존의 워드프로세서보다 훨씬 효율적인 작업이 가능합니다.

### 2.1 인터페이스 살펴보기

VS Code의 화면은 크게 5가지 영역으로 나뉩니다.

1.  **활동 바 (Activity Bar):** 화면 가장 왼쪽에 있는 좁은 세로 줄입니다. 탐색기, 검색, 소스 제어(Git), 확장 프로그램 등 주요 메뉴 아이콘이 위치합니다.
2.  **사이드바 (Side Bar):** 활동 바에서 선택한 메뉴의 상세 내용이 표시되는 영역입니다. 예를 들어 '탐색기'를 누르면 현재 폴더의 파일 목록이 보입니다.
3.  **편집기 (Editor):** 화면 중앙의 가장 큰 영역으로, 실제로 글을 쓰고 문서를 편집하는 공간입니다.
4.  **패널 (Panel):** 화면 하단에 위치하며, 문제(오류) 확인, 터미널(명령어 입력) 등의 기능을 수행합니다. (`Ctrl+J` 또는 `Cmd+J`로 열고 닫을 수 있습니다.)
5.  **상태 표시줄 (Status Bar):** 화면 최하단에 있으며, 현재 커서 위치, 파일 인코딩, Copilot 상태 등 정보를 표시합니다.

### 2.2 Markdown의 이해와 활용

행정 문서를 VS Code에서 작성할 때 가장 추천하는 형식은 **Markdown(마크다운)**입니다. Markdown은 복잡한 메뉴 클릭 없이 간단한 기호만으로 문서의 서식을 지정하는 방식입니다.

#### 2.2.1 왜 Markdown인가?
*   **집중력 향상:** 마우스로 글꼴이나 크기를 조절하는 데 시간을 뺏기지 않고, 오직 '내용' 작성에만 집중할 수 있습니다.
*   **범용성:** 작성된 문서는 PDF, HTML, Word 등 다양한 형식으로 쉽게 변환할 수 있습니다.
*   **AI 친화적:** GitHub Copilot은 Markdown 문법을 완벽하게 이해하므로, 문서의 구조를 파악하여 더 정확한 제안을 해줍니다.

#### 2.2.2 필수 문법 5가지
행정 문서 작성에 필요한 핵심 문법은 다음 5가지가 전부입니다.

1.  **제목 (Heading):** `#` 기호의 개수로 제목 수준을 정합니다.
    *   `# 대제목` (문서 제목)
    *   `## 중제목` (장 제목)
    *   `### 소제목` (절 제목)
2.  **목록 (List):** `-` 또는 `1.` 을 사용합니다.
    *   `- 항목 1` (순서 없는 목록)
    *   `1. 첫 번째` (순서 있는 목록)
3.  **강조 (Bold):** 강조하고 싶은 단어를 `**`로 감쌉니다.
    *   `**중요한 내용**` → **중요한 내용**
4.  **인용 (Quote):** `>` 기호를 사용하여 참고 문구나 규정을 인용합니다.
    *   `> 제1조(목적) 이 규정은...`
5.  **표 (Table):** `|` 기호로 칸을 나눕니다. (Copilot이 자동으로 만들어주므로 직접 타이핑할 일은 적습니다.)

### 2.3 유용한 확장 프로그램 추천

행정 업무 효율을 높여주는 필수 확장 프로그램을 소개합니다.

*   **Markdown All in One:** Markdown 작성에 필요한 단축키, 목차 자동 생성 등 편의 기능을 제공합니다.
*   **Markdown PDF:** 작성한 Markdown 문서를 클릭 한 번으로 깔끔한 PDF 파일로 변환해줍니다. 보고서 제출 시 매우 유용합니다.
*   **Korean Spell Checker (한국어 맞춤법 검사기):** 부산대학교 인공지능연구실의 맞춤법 검사기를 연동하여, 문서의 오탈자를 실시간으로 확인하고 교정해줍니다.

## 제3장: GitHub Copilot 핵심 기능

2025년 11월 현재, GitHub Copilot은 단순한 자동 완성을 넘어선 다양한 첨단 기능을 제공합니다.

### 3.1 채팅 (Chat): 대화형 AI 파트너

VS Code 사이드바의 채팅 아이콘을 클릭하거나 `Ctrl+I` (Mac: `Cmd+I`)를 눌러 인라인 채팅을 호출할 수 있습니다.

*   **사이드바 채팅:** "이번 달 학사 일정 정리해줘"와 같이 문서 전체나 프로젝트 전반에 걸친 질문을 할 때 사용합니다.
*   **인라인 채팅:** 문서 작성 중 특정 문단을 드래그하고 `Cmd+I`를 눌러 "이 문장을 좀 더 정중하게 다듬어줘"라고 요청할 때 사용합니다.

### 3.2 자동 완성 (Code Completion)

문서를 작성하다가 잠시 멈추면, Copilot이 회색 글씨(Ghost Text)로 다음에 올 내용을 제안합니다.

*   **수락:** `Tab` 키를 누르면 제안된 내용이 입력됩니다.
*   **거부:** `Esc` 키를 누르거나 계속 타이핑하면 제안이 사라집니다.
*   **다른 제안 보기:** `Alt + ]` (Mac: `Option + ]`)를 눌러 다른 문구 제안을 확인할 수 있습니다.

### 3.3 멀티 모델 활용 (Multi-Model Support)

이제 사용자는 작업의 성격에 따라 가장 적합한 AI 모델을 직접 선택할 수 있습니다. 채팅 창 상단의 모델 선택 메뉴에서 변경 가능합니다.

*   **GPT-5.1 (OpenAI):** 균형 잡힌 성능으로 일반적인 문서 작성, 이메일 초안 작성에 적합합니다.
*   **Gemini 3 Pro (Google):** 긴 문맥을 이해하는 능력이 탁월하여, 수백 페이지의 규정집을 참조하거나 대용량 데이터를 분석할 때 유리합니다.
*   **Claude 3.5 Sonnet (Anthropic):** 논리적 추론과 뉘앙스 파악에 강하여, 민감한 공문 작성이나 설득력이 필요한 기획안 작성에 추천합니다.

### 3.4 차기 편집 제안 (Next Edit Suggestions, NES)

2025년 새롭게 도입된 NES 기능은 사용자가 무엇을 수정하려는지 예측합니다. 예를 들어, 표의 첫 번째 행을 수정하면, 나머지 행들도 같은 패턴으로 수정해야 함을 인지하고, 커서를 이동하기도 전에 미리 수정안을 제안합니다. 단순 반복 수정 작업의 피로도를 획기적으로 줄여줍니다.

### 3.5 스마트 명령어

채팅 창에서 `/` 또는 `@`를 입력하여 특수한 기능을 사용할 수 있습니다.

*   **`@workspace`:** "현재 열려있는 프로젝트 전체"를 참조하라는 명령입니다. (예: `@workspace 작년도_사업계획서.md 파일을 참고해서 올해 예산안 초안 짜줘`)
*   **`/explain`:** 선택한 문단이나 복잡한 엑셀 수식을 알기 쉽게 설명해달라고 요청합니다.
*   **`/fix`:** 문법 오류나 오탈자를 수정해달라고 요청합니다.
*   **`/tone`:** (신규) 문체의 톤앤매너를 변경합니다. (예: `/tone formal` - 격식 있게, `/tone friendly` - 친근하게)

## 제4장: 실전 행정 업무 활용

이론을 넘어, 실제 대학 행정 현장에서 빈번하게 발생하는 업무 시나리오별 활용법을 알아봅니다.

### 4.1 이메일 및 공지사항 초안 작성

매번 비슷한 내용의 이메일을 쓰느라 고민할 필요가 없습니다. 핵심 키워드만 입력하면 Copilot이 격식 있는 문장으로 완성해줍니다.

**활용 예시:**
1.  VS Code에서 새 파일을 열고 다음과 같이 적습니다.
    ```markdown
    # 전체 교직원 대상 이메일
    - 주제: 2025학년도 하계 단축근무 시행 안내
    - 기간: 7월 1일 ~ 8월 15일
    - 시간: 10:00 ~ 16:00
    - 협조: 부서별 비상 연락망 유지 필수
    ```
2.  Copilot에게 요청합니다 (인라인 채팅 `Cmd+I`):
    > "위 내용을 바탕으로 정중하고 명확한 어조의 공지 이메일 초안을 작성해줘."
3.  결과물을 확인하고 필요한 부분만 살짝 수정하여 발송합니다.

### 4.2 회의록 요약 및 정리

받아적은 두서없는 회의 메모를 깔끔한 회의록으로 변환할 수 있습니다.

**활용 예시:**
1.  회의 중 속기한 메모 파일을 엽니다.
2.  사이드바 채팅에서 `@workspace` 명령어를 사용하여 요청합니다.
    > "@workspace 현재 파일의 회의 메모를 정리해서 1. 결정사항, 2. 향후 계획, 3. 담당자별 액션 아이템으로 요약된 공식 회의록 포맷으로 만들어줘."
3.  Copilot이 정리해준 내용을 새 파일(`회의록_20251120.md`)로 저장합니다.

### 4.3 엑셀 데이터 처리 및 수식 생성

복잡한 엑셀 함수를 외우지 않아도 됩니다. 원하는 계산 방식을 말로 설명하면 Copilot이 수식을 알려줍니다.

**활용 예시:**
1.  채팅창에 질문합니다.
    > "엑셀에서 A열에는 학생 이름, B열에는 중간고사 점수, C열에는 기말고사 점수가 있어. D열에 총점을 구하고, 총점이 90점 이상이면 'A', 80점 이상이면 'B', 나머지는 'C'로 표시하는 수식을 알려줘."
2.  Copilot이 제공한 수식(`=IF(A2+B2>=90, "A", IF(A2+B2>=80, "B", "C"))`)을 복사하여 엑셀에 붙여넣습니다.

### 4.4 다국어 문서 번역 및 교정

외국인 유학생이나 교원을 위한 영문 공지사항도 손쉽게 작성할 수 있습니다.

**활용 예시:**
1.  한글로 작성된 공지사항을 드래그하여 선택합니다.
2.  인라인 채팅(`Cmd+I`)을 열고 요청합니다.
    > "이 내용을 대학 행정에서 사용하는 공식적인 톤의 영어로 번역해줘."
3.  번역된 내용을 확인하고, 필요하다면 "조금 더 부드러운 표현으로 바꿔줘"라고 추가 요청하여 다듬습니다.

## 제5장: Git을 활용한 문서 버전 관리

Git을 사용하면 중요한 행정 문서를 안전하게 보관하고 관리할 수 있습니다.

### 5.1 저장소(Repository) 만들기

'저장소'는 프로젝트의 모든 파일과 변경 이력이 저장되는 공간입니다.

1.  VS Code에서 `파일 > 폴더 열기`를 통해 작업할 폴더(예: `2025_사업계획`)를 엽니다.
2.  왼쪽 활동 바에서 세 번째 아이콘(**소스 제어**)을 클릭합니다.
3.  **'리포지토리 초기화(Initialize Repository)'** 버튼을 클릭합니다. 이제 이 폴더는 Git의 관리를 받게 됩니다.

### 5.2 변경 사항 저장하기 (Commit)

문서를 수정하고 저장(`Ctrl+S`)했다고 해서 버전이 기록되는 것은 아닙니다. '커밋(Commit)'이라는 확정 행위를 해야 합니다.

1.  문서를 작성하거나 수정합니다.
2.  **소스 제어** 탭을 보면 변경된 파일 목록이 보입니다.
3.  파일 이름 옆의 `+` 버튼(변경 내용 스테이징)을 누릅니다. (장바구니에 담는 것과 같습니다.)
4.  위쪽 메시지 입력창에 **'무엇을 변경했는지'** 적습니다. (예: "1장 서론 초안 작성", "예산안 표 수정")
5.  **'커밋(Commit)'** 버튼(체크 표시)을 클릭합니다. 이제 현재 상태가 하나의 '버전'으로 영구 저장되었습니다.

### 5.3 과거 내역 확인 및 되돌리기

1.  **Git Graph** 확장 프로그램(추천)을 설치하면, 지하철 노선도처럼 변경 이력을 시각적으로 볼 수 있습니다.
2.  과거의 특정 시점을 클릭하고 **'Checkout'** 하면, 그 당시의 문서 상태로 파일 내용이 즉시 되돌아갑니다.
3.  실수로 내용을 지우고 저장했더라도, 직전 커밋으로 되돌리면 완벽하게 복구됩니다.

### 5.4 GitHub을 통한 협업 기초

내 컴퓨터(로컬)에만 저장하는 것이 아니라, 인터넷 상의 저장소인 **GitHub**에 업로드하면 팀원들과 공유할 수 있습니다.

1.  GitHub 웹사이트에서 `New Repository`를 생성합니다.
2.  VS Code의 소스 제어 탭에서 `...` 메뉴 > `원격(Remote)` > `원격 추가(Add Remote)`를 선택하고 GitHub 저장소 주소를 입력합니다.
3.  **'게시(Publish Branch)'** 또는 **'동기화(Sync Changes)'** 버튼을 누르면 내 컴퓨터의 작업 내용이 GitHub에 업로드됩니다.
4.  이제 팀원들은 이 주소를 통해 문서를 다운로드받고, 각자 수정한 내용을 합칠 수 있습니다.

### B. 문제 해결 (Troubleshooting)

**Q. Copilot 아이콘에 사선이 그어져 있고 작동하지 않아요.**
A. 로그인이 풀렸거나 라이선스가 만료되었을 수 있습니다. 아이콘을 클릭하여 다시 로그인하거나, GitHub 홈페이지에서 결제 정보를 확인하세요.

**Q. 한글 입력 시 제안이 너무 느려요.**
A. 인터넷 연결 상태를 확인하세요. Copilot은 클라우드 서버와 통신하므로 네트워크 속도에 영향을 받습니다. 또한, 문장을 끝까지 입력하지 말고 잠시 멈추면 제안이 더 잘 나옵니다.

**Q. 회사(대학) 보안 정책 때문에 GitHub을 쓸 수 없대요.**
A. 대학 정보전산원 등 IT 부서에 문의하여 'GitHub Copilot Business' 도입을 건의해보세요. 엔터프라이즈 버전은 데이터가 외부로 유출되지 않도록 하는 강력한 보안 기능을 제공하여 공공기관이나 대학에서도 안전하게 사용할 수 있습니다.
