# 문서 스타일 가이드 (Style Guide)

이 문서는 프로젝트 내 모든 마크다운(.md) 파일에 일관된 스타일을 적용하기 위한 규칙을 정의합니다.

## 1. 헤더 (Headers)

헤더는 논리적 계층 구조를 명확히 나타내기 위해 사용합니다.

- **`#` (H1):** 문서의 유일한 최상위 제목으로, 파일 당 한 번만 사용합니다.
- **`##` (H2):** 주요 섹션을 구분합니다.
- **`###` (H3):** 하위 섹션을 구분합니다.
- **`####` (H4):** 필요한 경우 더 세부적인 내용을 구분합니다.

계층 구조를 건너뛰지 마십시오. (예: `##` 다음에 `####`를 바로 사용 금지)

## 2. 마크다운 문법 표준 (Markdown Syntax)

- **목록 (Lists):**
    - 순서가 없는 목록은 하이픈(`-`) 또는 별표(`*`)를 사용합니다.
    - 들여쓰기는 스페이스 4칸을 사용합니다.
- **강조 (Emphasis):**
    - `**굵게**` (Bold)는 중요한 용어나 문장을 강조할 때 사용합니다.
    - `*기울임*` (Italic)은 보조적인 설명이나 미묘한 강조에 사용합니다.
- **인용 (Blockquotes):** `>`를 사용하여 인용구나 팁을 표현합니다.

## 3. 코드 블록 (Code Blocks)

모든 코드 블록에는 언어 식별자를 명시하여 구문 강조(Syntax Highlighting)가 적용되도록 합니다.

**좋은 예:**
```bash
# 명령어 예시
echo "Hello, World!"
```

**나쁜 예:**
```
명령어 예시
echo "Hello, World!"
```

## 4. 이모지 사용 규칙 (Emoji Guidelines)

이모지는 독자의 주의를 환기시키거나, 특정 정보의 성격을 시각적으로 전달하는 데 효과적입니다. 과도한 사용은 자제하고, 아래의 규칙에 따라 사용합니다.

- `💡` (Tip): 유용한 팁이나 아이디어를 제공할 때
- `⚠️` (Warning): 주의가 필요한 중요한 정보
- `✅` (Success/Check): 완료된 항목이나 성공적인 결과를 나타낼 때
- `❌` (Failure/Error): 실패한 항목이나 오류를 나타낼 때
- `📝` (Note): 추가적인 메모나 설명을 덧붙일 때

## 5. 학습 진도 체크박스 (Progress Checkbox)

사용자의 학습 편의를 위해, 각 문서의 목차에는 GitHub 마크다운 체크박스를 사용합니다.

- **문법:** `- [ ]`
- **예시:** `- [ ] [1. 나만의 디지털 집무실 만들기](./level-1-basics/01-setup.md)`
