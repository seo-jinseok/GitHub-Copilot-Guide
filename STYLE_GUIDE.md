# 문서 스타일 가이드
# (Style Guide)

프로젝트 내 모든 마크다운(.md) 파일에 적용하는 스타일 규칙입니다.

## 1. 문체 원칙

| 항목 | 규칙 |
|------|------|
| **톤** | 간결하고 전문적인 매뉴얼 톤 |
| **경어** | '하십시오' 또는 '해요' 체 |
| **이모지** | 사용하지 않음 |
| **설명** | 불필요한 서론/결론 최소화, 핵심 정보 중심 |

## 2. 헤더 규칙

| 레벨 | 용도 | 규칙 |
|------|------|------|
| `#` (H1) | 문서 제목 | 파일당 1회만 사용 |
| `##` (H2) | 주요 섹션 | - |
| `###` (H3) | 하위 섹션 | - |
| `####` (H4) | 세부 항목 | 필요시에만 사용 |

계층 구조를 건너뛰지 않습니다. (예: `##` → `####` 금지)

## 3. 마크다운 문법

| 요소 | 규칙 |
|------|------|
| **목록** | `-` 또는 `*` 사용, 들여쓰기 4칸 |
| **강조** | `**굵게**` (중요 용어), `*기울임*` (보조 설명) |
| **인용** | `>` 사용 (팁, 주의사항) |
| **표** | 정보 전달 시 표 우선 사용 |

## 4. 코드 블록

언어 식별자를 반드시 명시합니다.

```bash
# 올바른 예시
echo "Hello, World!"
```

## 5. 다국어 병기

한국어 먼저 작성 후 줄바꿈, 괄호 안에 영어를 작성합니다. 동일한 스타일(헤더, 볼드 등)을 적용합니다.

```markdown
## 사용자 경험
## (User Experience)
```

## 6. 학습 진도 체크박스

목차에 GitHub 마크다운 체크박스를 사용합니다.

```markdown
- [ ] [1. 설정](./01-setup.md)
- [ ] [2. 기본 사용법](./02-basics.md)
```
